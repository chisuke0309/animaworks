## 完了した作業

### 1. sora の画像生成修正
- **問題**: sora が `generate_tiktok_image` ツール（存在しない）を呼び出してエラー
- **対応**:
  - `fal-client==0.13.1` + `pillow==12.1.1` を venv にインストール
  - `/Users/chisuke/.animaworks/scripts/generate_image.py` を新規作成（fal.ai FLUX使用）
  - `~/.animaworks/animas/sora/injection.md` を更新（Bash + スクリプト呼び出しに変更）
- **結果**: sora が `slide_1.png`（TikTokカルーセル背景）の生成に成功 ✓

### 2. rue/kuro を Gemini 3.1 Flash-Lite Preview に移行
- **理由**: qwen3.5:9b-8k（32Kコンテキスト）はシステムプロンプト肥大化でツール履歴が切れる
- **対応手順**:
  1. `GOOGLE_API_KEY=REDACTED_GOOGLE_API_KEY` を `.env` に追加
  2. `credentials.google.api_key` を `config.json` に追加（**重要**: これがないとAnthropicキーがGemini APIに送られる）
  3. `animaworks anima set-model rue/kuro gemini/gemini-3.1-flash-lite-preview --credential google`
  4. `animaworks stop && animaworks start`（サーバーデーモン再起動必須）
- **結果**: 両者 `retries=0, response_len>0` で正常稼働 ✓
- **コンテキスト**: 32K → 1,000K に拡大、TIER_STANDARD問題が解消

### 3. Ollama 削除
- モデル `qwen3.5:9b-8k`（6.6GB）+ `qwen3.5:9b`（6.6GB）削除
- Ollama本体 + mlx/mlx-c（計185MB）を `brew uninstall` で削除
- `~/.ollama` ディレクトリ削除
- 合計 **約13GB** 解放

## 重要な設定・ファイル

### 新規作成ファイル
- `/Users/chisuke/.animaworks/scripts/generate_image.py` — fal.ai FLUX 画像生成スクリプト
  - 使い方: `FAL_KEY=$FAL_KEY $PYTHON $SCRIPT --prompt "..." --ratio 9_16 --output /path/to/out.png`
  - ratio: `9_16` / `portrait_4_3` / `square`

### 更新ファイル
- `~/.animaworks/animas/sora/injection.md` — generate_image.py の Bash 呼び出し手順
- `/Users/chisuke/Projects/animaworks/.env` — `GOOGLE_API_KEY` 追加
- `~/.animaworks/config.json` — `credentials.google.api_key` 追加

### 重要な知見
- **モデル変更時は必ず `--credential` を指定する**: デフォルト credential は `anthropic`、Gemini使用時は `google` 必須
- **新しいenv変数はサーバーデーモン再起動が必要**: `animaworks restart anima` では引き継がれない
- **cicciへの情報共有**: `animaworks send chisuke cicchi "..."` で随時送れる
