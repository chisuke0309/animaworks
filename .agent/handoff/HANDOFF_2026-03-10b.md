## ✅ 今セッションの完了作業

### 1. メッセージ即時読み取り遅延の原因解明
**問題**: cicchiがrue/kuroのメッセージをすぐ読まない
**根本原因**: `msg_heartbeat_cooldown_s: 300`（5分クールダウン）
- メッセージトリガーまたはスケジュールハートビート後300秒以内は新着メッセージを遅延処理
- `lifecycle.py` の `_inbox_watcher_loop` → `_is_in_cooldown` → `_schedule_deferred_trigger` がメカニズム
- `intent="report"` は `actionable_intents` に含まれているので intent フィルタは問題なし
- 対策: テスト時は `config.json` の `msg_heartbeat_cooldown_s` を `60` に下げれば速くなる（要検討）
- 自動パイプライン（07:00 cron）では cronがheartbeatを更新しないため、実用上は問題なし

### 2. Telegram通知の設定

  - `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` は `/Users/chisuke/Projects/animaworks/.env` に設定済み

### 2. クールダウン（インタラクティブテスト時の遅延）
- **状況**: `msg_heartbeat_cooldown_s: 300` のため、テスト中に「rue/kuroの返信が来ても cicchi が5分待つ」ことがある
- **対応**: テスト時は一時的に `60` に下げても良いが、カスケード防止のトレードオフあり
